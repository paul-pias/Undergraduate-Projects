1. Introduction


ForRent.com is a database for apartments/rooms available for rent in different areas of the city. It is the only solution of its kind available in Bangladesh that allows users to look for apartments as well as rooms that bachelors can share. ForRent.com allows users to put their flats for rent and as well as to search for the perfect place to live. Here the User or owner of apartments as well as the manager of a bachelor apartment can give their apartment or room details, so that people who are in search of a place to live can check if that apartment or room is suitable for them. The website is open for all to register free of cost and registered users can edit or even delete their apartment information whenever they want by logging into their accounts. ForRent.com makes conventional house renting procedures simple and seamless.

1.1 Purpose of this Document

This SDS document is intended to cover the technical aspects of the development of ForRent.com.

1.2 Scope of the development project

4. User Interface Issues

The Home Page will feature the listings from the Database and the Search Widget. Users can use the Search Widget to classify their location, budget, type of listing, number of bedrooms, etc.

Users will have the option to sign up using Facebook or Google in addition to the conventional form.

The View Details page will show users all the facilities available. Users will also be able to view the exact location on the map. Web Users will get recommendations of similar listings if they allow cookies. App users will get recommendations according to their search data preferences.



5. Execution Architechture

The front-end of ForRent.com will be made using jQuery and a bootstrap template of a typical e-commerce site and a database will be made in phpmyadmin, which is a web-based tool for administering MariaDB and MySQL. "MariaDB is one of the most popular Open Source relational databases. We intend to use MariaDB because it is fast, scalable and robust, with a rich ecosystem of storage engines, plugins and many other tools which make it very versatile for a wide variety of use cases."[1]

The front-end and the back-end will then be connected using the CodeIgniter framework, which utilizes the MVC pattern. It helps to connect the front-end and back-end of a website or web application by organising the codes into three interconnected parts. The database is managed in the Model section, the front-end code is managed by the View section and finally the php code that connects the database to the front-end is managed by the Controller section.The resulting application will then hosted be on a server. Initial testing will be done by hosting on a XAMPP Local Server.

We will also try to utilize the real-time database of Firebase. This can aid in the development of the Android app.



6. Design Decisions & Tradeoffs

Registered users will have the option to add either Apartments or Rooms in the "Add Listings" page. We chose not to create two types of users who can either add only Apartments or only Rooms. This is done for overall simplicity, as separating the features for 2 types of users may cause some users to create 2 types of accounts for adding rooms and apartments. Unifying this feature addresses a larger userbase and makes it easier for users to maintain their account.
